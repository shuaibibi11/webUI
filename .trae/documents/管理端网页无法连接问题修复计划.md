# 管理端网页无法连接问题修复计划

## 问题分析
1. 管理端网页 http://192.168.3.52:13083/ 显示"无法连接"
2. 连接被拒绝，说明端口13083没有服务在监听
3. admin-web进程未运行
4. admin-web配置的是端口13080，而非13083
5. 端口13083未被任何服务占用

## 修复步骤

### 1. 启动admin-web服务
- 进入admin-web目录：`cd /data/webUI-java/admin-web`
- 启动服务：`npm run dev`
- 检查服务启动日志，确认运行状态和端口

### 2. 验证服务可访问性
- 使用curl测试本地访问：`curl -v http://localhost:13080/`
- 如果服务在其他端口启动（如13083），测试对应端口
- 检查是否能正常获取HTML响应

### 3. 配置正确的端口
- 如果需要固定在13083端口，修改vite.config.ts中的port配置
- 重启服务使配置生效
- 验证服务在13083端口运行

### 4. 测试外部访问
- 使用curl测试外部访问：`curl -v http://192.168.3.52:13083/`
- 检查是否能正常获取HTML响应

### 5. 连续暴力测试
- 执行5次连续的curl请求，验证服务稳定性
- 检查每次请求的响应状态码和内容
- 确保没有连接中断或加载失败情况

### 6. 检查依赖服务
- 确保admin-api服务正常运行
- 验证admin-web与admin-api的通信是否正常
- 检查API代理配置是否正确

## 预期结果
1. admin-web服务成功启动，在端口13083运行
2. 能够正常访问 http://192.168.3.52:13083/
3. 连续5次测试全部通过，无连接中断或加载失败
4. 管理端网页能够正常显示和使用

## 注意事项
- 确保服务启动时没有端口冲突
- 检查防火墙设置，确保端口13083开放
- 验证CORS配置，确保外部请求能够正常处理
- 检查日志文件，及时发现和解决问题
